🧱 1. Task Management Entities

🔧 These define the core complaint resolution flow for solvers.
Entity	Description
Task	A job generated from a complaint and assigned to a solver
TaskAssignment	Links solver(s) to a task, optionally includes assigned_by and due_date
TaskUpdateSubmission (or TaskLog)	Updates made by the solver (status, notes, timestamps)
TaskMedia	Uploaded photos/videos from solver as part of update submission
Complaint	Original citizen complaint — includes type, location, etc. (read-only for solver)
📣 2. Communication & Feedback Entities

💬 Manage how solvers receive info and how citizens give feedback.
Entity	Description
Notification	Alerts sent to solver (task assigned, reminders, admin messages)
CitizenRating	Star or numeric rating given by citizen after task is marked resolved
CitizenComment	Optional detailed comment/feedback from citizen (can be tied to rating)
👤 3. Other (Profile, System, Reward, Payments)

🧍 Entities supporting solver lifecycle and engagement.
Entity	Description
User	Core user data (username, email, role: 'solver')
Profile	Extended fields like zone, skills, phone, avatar
Reward	Track earned badges or points for performance
Availability (optional)	Solver availability flag or working hours
PaymentRequest	Solver's request to admin for payment related to completed tasks
